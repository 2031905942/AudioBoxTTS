## Context

当前项目在“项目 -> 声音库”子面板中提供“同步声音库”按钮。按钮实际触发的是文件级同步：将 Wwise 工程 `GeneratedSoundBanks` 中的产物与 Unity 工程的声音库目录进行对比，然后对 Unity 目录执行复制/覆盖/删除等操作。

现状问题：按钮本身缺少对“会做什么”的说明，用户在点击前无法理解其影响范围（哪些目录、哪些文件类型、是否会删除/覆盖），导致误操作风险较高。

约束：
- 不改变现有同步逻辑与结果，只补齐可见说明（tips）。
- 文案需与当前实现保持一致（覆盖/删除行为、语言选择影响、Wwise 版本差异等）。

## Goals / Non-Goals

**Goals:**
- 在“同步声音库”按钮上提供清晰的 tips，解释该操作的行为与影响范围。
- tips 体现关键行为：对比源/目标目录、复制新增、同步（覆盖）变更、删除多余、清理空目录。
- tips 体现关键范围：`.bnk/.wem`、元数据（`SoundbanksInfo/PlatformInfo/PluginInfo` 的 `json/xml`）、Wwise 2022+ 的 streamed media（`Media/`）、以及外部源 `.wwopus`。
- tips 说明“语言列表”勾选对同步范围的影响。
- 同步完成后向用户输出本次同步的具体变更明细（新增/更新/删除的文件列表），便于核对。

**Non-Goals:**
- 不引入新的 UI 页面/对话框/流程（仅使用现有 tooltip/tips 机制）。
- 不在同步前做额外的扫描/统计/预览（避免引入耗时与复杂度）。
- 不修改同步算法、文件过滤规则或任务调度逻辑。

## Decisions

- **使用 Qt/qfluentwidgets 的 tooltip（`setToolTip`）承载 tips 文案**：实现成本最低，符合“仅补说明”的范围。
  - 备选：使用 `TeachingTip`（更强展示能力）但引入额外锚点/显示时机判断，超出本次最小改动范围。

- **完成后采用“摘要 + 详细报告窗口”**：InfoBar 只展示一行摘要（新增/更新/删除数量），同时弹出一个可复制的文本窗口展示详细文件列表。
  - 原因：InfoBar 适合短提示，不适合承载长列表；文本窗口便于一目了然查看与复制到提交说明。
  - 约束：不改变同步逻辑，仅改变结果展示方式。

- **tips 文案保持“高层行为 + 关键例外/范围”**：不包含动态统计（例如将要复制/删除数量），避免需要在点击前执行 I/O 扫描。
  - 同时明确“不会生成 SoundBank，只同步已有产物”，降低用户误解。

- **文案与实现保持可追溯的一致性**：tips 的条目来源于当前同步实现中的关键路径（语言目录、Wwise 2022+ `Media/`、`.wwopus` 处理、删除行为）。
  - 风险：未来同步实现变化导致 tips 过期；通过将文案聚焦在稳定行为（对比/复制/覆盖/删除）并在任务中加入“文案与实现对齐检查”来缓解。

## Risks / Trade-offs

- [tips 与实现不一致] → 保持文案描述为“行为类别”而非过细路径；实现变动时同步更新文案。
- [tooltip 过长影响可读性] → 采用短句 + 分行/列表化表达，必要时压缩为 5-8 行关键点。
- [变更明细过长导致阅读困难] → 对列表做合理截断（例如每类最多展示 N 项），并显示“还有 X 项未展示”。
- [用户以为会在点击时生成 SoundBank] → 文案明确“该操作同步文件，不负责在 Wwise 里生成 SoundBank”。

## Migration Plan

- 随代码变更一起发布，无需数据迁移。
- 回滚策略：回退 UI 文案变更即可，不影响同步功能本身。

## Open Questions

- tips 采用纯文本换行还是富文本（HTML）展示，是否需要统一风格（与其他 tooltip 一致）。默认先用纯文本换行实现。
