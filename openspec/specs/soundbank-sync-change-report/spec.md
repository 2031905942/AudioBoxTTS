## ADDED Requirements

### Requirement: 同步完成后展示本次变更明细
系统 MUST 在“同步声音库”任务完成后，向用户输出本次同步的具体变更明细，使用户能够一目了然地看到发生了哪些更改。

输出内容 MUST 至少包含以下三类的文件列表（以 Unity 声音库目录的相对路径表示）：
- 新增文件（本次从 Wwise 侧复制到 Unity 侧的文件）
- 更新文件（本次从 Wwise 侧覆盖更新到 Unity 侧的文件，仅列出内容发生变化的文件）
- 删除文件（本次从 Unity 侧删除的多余文件）

#### Scenario: 同步完成后可见变更清单
- **WHEN** 用户点击“同步声音库”并等待任务完成
- **THEN** 系统展示本次同步的变更明细，且新增/更新/删除三类文件清单清晰可读

### Requirement: 变更明细支持长列表的可读性
当单次同步涉及大量文件时，系统 MUST 保证输出仍然可读。

系统 SHOULD 对每类列表做合理截断（例如每类最多展示 N 项），并在截断时提示“还有 X 项未显示”。

#### Scenario: 大量文件时显示截断提示
- **WHEN** 本次同步的新增/更新/删除文件数量超过单次展示上限
- **THEN** 系统在对应列表末尾提示仍有未展示项的数量
